High-Level Goals

For IMIchat's redesign, I created all the persona for customers, agents and their managers. Following that, I set following 3 high level goals that are valid for any feature

  1. Make it efficient and fast for agents to handle high volumes of queries
  2. Mistakes from agents should have minimum reflection on end customers. That is, system should be able to tackle agents’ mistakes.
  3. Make it easier for team managers to monitor the situation and manage their teams.

What is 'Pick Chat Feature/Flow'

This feature is analogous to real life offices. If you are working with other colleagues and have hundreds of files to finish, everyone will take 4-5 files each and will start working. As soon as anyone finishes off their files, he/she will pick up more files from the common stack.
This is exactly what happens in contact centres. Customer queries lands in common queue and each agent picks up queries in his/her queue to solve it. Picking up queries from common queue is “Chat pickup feature’.

Few Other Considerations

As our data suggested that users had problems in understanding few existing features, we laid down following guidelines:

  1. Make learning curve steep. Therefore, reduce dependency and save money on training.
  2. Make users use the platform in the most optimal way. Features should be self-explanatory so that no one should use the platform in any other way other than we thought of them.

Micro-Interactions

Micro-interactions play a very important role in overall understanding and usability of the product. I give a lot of care how animation and states have to be while designing any product/feature.
As animation provides feedback to the user, therefore animation has the power to reduce errors by user as well. For our features, I follow guidelines adapted from NN group's research of feedback.

Release and Product Directions

Making huge changes in the platform which is used for an average of 8hrs/day by thousands of users will result in catastrophe for customer support, we planned to make changes in phases.
